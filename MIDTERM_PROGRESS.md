# Job Tech Analyzer 중간 개발 현황

## 1) 연구/개발 목표
- 채용공고 텍스트를 기반으로 직무별 요구 기술과 우대 기술을 추출해 기술 스택 트렌드를 분석한다.
- 사용자가 직무를 검색하면 핵심 기술 통계와 데이터 출처를 직관적으로 확인할 수 있는 웹 서비스를 제공한다.
- 사용자 보유 기술과 직무 요구 기술을 비교해 개인화된 스킬 갭 정보를 제공한다.

## 2) 현재까지 구현 범위

### 2-1. 데이터 수집 파이프라인
- Greenhouse, Lever API를 활용해 채용공고 데이터를 수집하는 스크립트를 구현했다.
- 공고 본문 HTML을 제거/정규화하여 분석 가능한 텍스트로 전처리한다.
- 수집 결과를 CSV(`data/appsflyer_jobs.csv`) 형태로 저장한다.

### 2-2. 분석 엔진
- 직무명 기준 공고 필터링 및 직무 후보 추출 로직을 구현했다.
- 질의어 동의어 정규화(예: backend/back-end/백엔드)로 검색 일관성을 높였다.
- 기술 사전(`tech_stack.txt`) 기반으로 요구/우대 기술을 추출한다.
- 기술별 등장 횟수와 비율(%)을 계산해 통계 결과를 생성한다.

### 2-3. 웹 서비스(MVP)
- FastAPI 기반 웹 인터페이스를 구현했다.
- 검색 결과가 다중 후보일 때 후보 선택 흐름을 제공한다.
- 요구 기술/우대 기술 표를 화면에 출력한다.
- 회사-플랫폼별 공고 수 및 공고 목록(일부)을 함께 시각화한다.

### 2-4. 개인화 기능(1차)
- 사용자 보유 스킬 입력 기능을 구현했다.
- 상위 요구 기술 기준 매칭률(%)과 부족 스킬 목록을 계산한다.

## 3) 구현 완료 수준 요약
- 핵심 파이프라인(수집 -> 전처리 -> 분석 -> 시각화) 동작 확인 완료
- 현재 단계는 정확도 고도화 및 데이터 관리 구조 개선(DB 도입 검토) 전 단계
